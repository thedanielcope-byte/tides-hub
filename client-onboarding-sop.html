Client Onboarding SOP

Standard Operating Procedure · Client Portal Setup
Tides Bookkeeping LLC
Document Version 1.0
Last updated: May 2026

Purpose

This procedure documents how to set up a new client portal at portal.tidesbookkeeping.com. Following this SOP ensures secure, consistent onboarding in approximately 5–10 minutes per client. Each client receives a private, password-less hub for accessing financial reports, uploading documents, and quick-linking to their dashboards.

Tools Required

  • Microsoft OneDrive (Tides admin account)
  • Cloudflare Zero Trust dashboard access (admin)
  • Tides Hub (hub.tidesbookkeeping.com/client-onboarding.html)
  • Terminal access on admin computer (for git push)

Estimated Time

5–10 minutes per client (after first-time setup is complete).

1 Set up OneDrive folders

Create one top-level folder per client. Use the standard structure below — consistency makes future audits and team handoffs easier.

📁 Tides Bookkeeping Clients/ 📁 [Client Name]/ ← top-level for this client 📁 Reports/ ← monthly P&L, balance sheet, cash flow 📁 2026/ January/, February/, March/... 📁 Uploads/ ← client drops receipts/invoices here 📁 Payroll/ ← optional — only if you do their payroll 📁 Tax Documents/ ← year-end packages, 1099s, etc.
Pro tip: Maintain an empty Client Template/ folder with this structure pre-built. When onboarding a new client, right-click → Copy → Rename. Saves ~2 minutes per client.

2 Grab share links from each folder

Two distinct share types are required, depending on whether the client should view files or upload them. Getting this wrong is a security risk — clients could potentially modify or see others' files.

Folder How to share Type Used as
Reports (parent only — one URL covers all months) Right-click folder → Share → set "Anyone with the link can view" View only reports_url
Uploads Right-click folder → Request Files → set title: "Upload to Tides" Upload only upload_url
Payroll (if applicable) Right-click folder → Share → "Anyone with the link can view" View only payroll_url
Tax Documents Right-click folder → Share → "Anyone with the link can view" View only Quick Link
Why two share types? A standard share link allows the client to view (download) files but not modify or upload. A Request Files link allows the client to upload only — they cannot see other files in the folder. This prevents one client from accidentally seeing another client's documents.

Other URLs to Gather

  • QuickBooks Online URL — log into QBO, copy the URL of their company dashboard. Use https://qbo.intuit.com/app/homepage as fallback.
  • Puzzle.io Dashboard URL — log into Puzzle, copy their workspace URL.
  • Optional extras: bank portal, insurance portal, internal docs — any URL the client should have quick access to.

3 Add the client's email to Cloudflare Access

This grants the client permission to log in. Without this step, the client cannot access their portal — even with the correct email.

  1. Go to Cloudflare Dashboard → Zero Trust → Access → Policies
  2. Click Allow Clients policy → Edit
  3. Add the client's email address as an allowed identity
  4. Save changes

4 Generate the client hub via the onboarding form

Use the interactive form at hub.tidesbookkeeping.com/client-onboarding.html to compile all the client information. The form auto-saves your progress and generates a complete prompt for Claude.

  1. Open CLIENTS → New Client Onboarding in the Tides Hub
  2. Fill in client identity (name, slug, email)
  3. Paste the OneDrive share links you gathered in Step 2
  4. Add Quick Link URLs (QuickBooks, Puzzle, any extras)
  5. Click 📋 Copy Claude Prompt
  6. Paste into a Claude conversation

Claude will update the client database (_scripts/clients.csv), generate the client's hub HTML, and update the routing map automatically.

5 Push the changes to deploy

Run the following commands in Terminal:

cd "/Users/danielcope/Library/Mobile Documents/com~apple~CloudDocs/Claude/tides-portal" git add -A git commit -m "Add client: [Client Name]" git push

Cloudflare Pages auto-deploys within 2 minutes.

6 Send the welcome email

Send the client this email template (substitute their information):

Subject: Your Tides Bookkeeping Portal is Ready Hi [First Name], Your secure client portal is now live at: https://portal.tidesbookkeeping.com To sign in, just enter your email ([their email]). We'll send you a 6-digit code — no password to remember. Inside your portal you'll find: • Monthly financial reports (P&L, balance sheet, cash flow) • Secure document upload (for receipts, invoices, etc.) • Quick links to your dashboards and tax documents • Direct chat with your bookkeeper Let us know if you need anything! — Tides Bookkeeping

7 Verify (recommended)

Before notifying the client, verify the setup yourself:

  1. Open https://portal.tidesbookkeeping.com in an incognito browser
  2. Enter the client's email address
  3. Confirm a code arrives at the client's email (check with them)
  4. Enter the code → confirm it routes to /clients/[their-slug]/
  5. Click each button (Reports, Upload, Quick Links, Email Bookkeeper) to confirm all links work

Troubleshooting

Client doesn't receive a login code

  • Verify their email is in the Cloudflare Access "Allow Clients" policy
  • Ask them to check spam / junk folder for emails from noreply@notify.cloudflare.com
  • Check Cloudflare service status at cloudflarestatus.com

Client logs in but sees "No portal configured"

  • Confirm the email in _scripts/clients.csv matches exactly what the client typed
  • Confirm git push completed successfully
  • Confirm Cloudflare Pages deployment status is "Success" in the dashboard

Reports / Upload links don't work

  • Open the OneDrive share link directly in incognito to verify it's still active
  • Check OneDrive sharing permissions haven't expired or been revoked

Offboarding a Client

When a client relationship ends:

  1. Remove their email from the Cloudflare Access "Allow Clients" policy (this immediately revokes access)
  2. Remove their row from _scripts/clients.csv
  3. Run python3 _scripts/add-client.py --csv to regenerate the routing map
  4. Optionally archive their OneDrive folder (move to Archived Clients/)
  5. Commit and push
