1 Set up OneDrive folders

Create one top-level folder per client with this structure:

📁 Tides Bookkeeping Clients/ 📁 [Client Name]/ ← top-level for this client 📁 Reports/ ← monthly P&L, balance sheet, cash flow 📁 2026/ January/, February/, March/... 📁 Uploads/ ← client drops receipts/invoices here 📁 Payroll/ ← optional — only if you do their payroll 📁 Tax Documents/ ← year-end packages, 1099s, etc.

Pro tip: Keep an empty Client Template/ folder. When onboarding new clients, right-click → copy the template, rename it. Saves 2 min of folder creation each time.

2 Grab share links from each folder

Two different share types matter — get this right or clients can see/edit things they shouldn't:

Folder How to share Type Used as
Reports (parent folder only — one URL covers all months) Right-click → Share → "Anyone with link can view" View only reports_url
Uploads Right-click → Request Files → title: "Upload to Tides" Upload only upload_url
Payroll (optional) Right-click → Share → "Anyone with link can view" View only payroll_url
Tax Documents Right-click → Share → "Anyone with link can view" View only Quick Link

Why two share types? Regular share = client can VIEW (download) but not modify. Request Files = client can UPLOAD but can't see other files in the folder. Critical for security: prevents one client from accidentally seeing another's docs.

4 After Claude generates the hub

Claude will reply with a confirmation. Then run these commands in Terminal:

cd "/Users/danielcope/Library/Mobile Documents/com~apple~CloudDocs/Claude/tides-portal" git add -A git commit -m "Add client: [Client Name]" git push

Cloudflare Access: Don't forget to add the client's email to the allowlist policy:

  • Cloudflare Dashboard → Zero Trust → Access → Policies → Allow Clients → Edit
  • Add email → Save

Email the client: Send them their portal URL: https://portal.tidesbookkeeping.com — they'll enter their email, get a 6-digit code, and land on their hub.
